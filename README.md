# Shade System

Shade System is an academic full-stack web project developed to help manage shaded outdoor areas on campus in a more organized, visual, and efficient way.

The system allows users to view campus areas, upload area maps, place shade elements on top of the map, edit their properties, and manage the information through a centralized platform.

## Project Overview

Many campuses manage shade-related information in a fragmented way using spreadsheets, images, messages, and manual updates.  
Shade System was designed to provide a single organized platform for managing shaded areas, improving visibility, coordination, and decision-making.

## Main Features

- User login and sign up
- Area management
- Upload area maps
- Add, edit, and delete shade elements
- Interactive area editing
- Role-based access (Admin / Viewer)
- Responsive interface for desktop and mobile use

## System Architecture

The project is built using a client-server architecture:

- **Client** – responsible for the user interface and user interaction
- **Server** – handles API requests, business logic, authentication, and data flow
- **Database** – stores users, areas, and shade information

## Database

The project includes SQL files for the main database entities:

- Users
- Areas
- Shades

These files are located under the `server/database/` directory.
